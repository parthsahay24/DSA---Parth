# 🤝 Contributing to DSA---Parth

Thanks for your interest in contributing! 🚀  
This repository is focused on **Data Structures & Algorithms practice**, so contributions should follow structured and clean standards.

---

## 📌 Contribution Rules

### 1. 🧠 Problem Solving Guidelines
- Write **clean and readable code**
- Prefer **optimal solutions** (mention time & space complexity)
- Avoid unnecessary libraries
- Use **standard C++ headers** (no `bits/stdc++.h`)

---

### 2. 📝 Code Format
Every solution must follow this structure:

```cpp
// Problem: <Problem Name>
// Author: <Your Name>
// Approach: <Short explanation>
// Time Complexity: O()
// Space Complexity: O()

#include <iostream>
using namespace std;

int main() {
    // your code
}
```
---

### 3. 📂 Folder Structure
- Place files in correct folders:
  - `Basic DSA/`
  - `Advance DSA/`
- Use meaningful file names:
  - `two_sum.cpp`
  - `binary_search.cpp`
  - `merge_sort.cpp`

---

### 4. 🌿 Branching Rules
- Never push directly to `main`
- Create a new branch:
  - `feature/problem-name`
  - `fix/issue-name`

---

### 5. 🔁 Pull Request Rules
- Link the issue in PR:
  - `Fixes #<issue_number>`
- Add proper title:
  - `Added Binary Search implementation`
- Keep PR **small and focused**

---

### 6. 💬 Code Review Requirements
- Add your name as a comment in code
- Follow suggestions from reviewers or AI tools (CodeRabbit)
- Ensure code compiles without errors

---

### 7. 🚫 What NOT to do
- Don’t submit copied code without understanding
- Don’t use unnecessary complexity
- Don’t push unrelated changes in one PR

---

### 8. ✅ Before Submitting PR
- Code compiles successfully
- Follows format
- No unnecessary files (`.dSYM`, executables, `.vscode`)
- Proper naming

---

## 🏁 Goal of this Repo
To build a **clean, structured, beginner-friendly DSA resource** for everyone.

---

## 🙌 Happy Coding!
If you improve even a small thing, you’ve contributed. 🚀
